## Quick Start

### Enabling Demo Mode

1. **Navigate to Demo Control Panel**
   ```
   Open app ‚Üí Navigate to /demo route
   Or add a settings button to access demo mode
   ```

2. **Toggle Demo Mode ON**
   - Flip the switch to enable demo mode
   - App will automatically seed 25 realistic check-ins
   - Demo data includes 6 users, 8 spots

3. **Start Filming!**
   - Feed will now show populated content
   - All interactions are realistic and smooth
   - Ready for professional demos

### Disabling Demo Mode

1. **Toggle Demo Mode OFF**
   - All demo data will be cleared
   - App returns to fresh state
   - Ready for real user testing

## Best Practices

### Before Filming

- ‚úÖ **Clean Build**: Run on physical device, not simulator
- ‚úÖ **Good Lighting**: Natural light or softbox for screen recording
- ‚úÖ **Hide Status Bar**: Use iOS screen recording (swipe up control center)
- ‚úÖ **Disable Notifications**: Turn on Do Not Disturb
- ‚úÖ **Charge Device**: 80%+ battery for smooth performance
- ‚úÖ **Clear Cache**: Fresh app install for cleanest demo

### During Filming

- üéØ **Smooth Interactions**: Tap deliberately, not frantically
- üéØ **Show Animations**: Pause to let entrance animations complete
- üéØ **Highlight Features**: Use slow scrolls to show polish
- üéØ **Haptic Moments**: Visible press feedback on buttons
- üéØ **Natural Pacing**: Don't rush - let UI breathe

### After Filming

- üìπ **Edit for Impact**: Cut long pauses, speed up scrolling
- üìπ **Add Voiceover**: Explain features as they appear
- üìπ **Overlay Text**: Highlight key features with labels
- üìπ **Music**: Upbeat background music (Linear, Notion vibes)
- üìπ **Export Quality**: 1080p minimum, 60fps preferred

## Pro Tips

### Showcasing Polish

1. **Entrance Animations**
   - Feed items slide in with staggered delays (50ms)
   - Cards scale up with spring physics
   - Icons bounce on first appearance

2. **Haptic Feedback**
   - Buttons pulse on press
   - Visible scale transform (0.98x)
   - Quick, satisfying interactions

3. **Loading States**
   - Skeleton loaders with shimmer effect
   - No jarring content pops
   - Smooth transitions

4. **Empty States**
   - Illustrated with helpful CTAs
   - Friendly copy, never blank screens
   - Guide users to next action

## Troubleshooting

### Demo Mode Won't Enable
- Check if `/demo` route is accessible
- Verify demo mode service is imported
- Check console for errors

### Demo Data Not Appearing
- Toggle demo mode OFF then ON
- Check async storage permissions
- Verify seedDemoFeed() completed

### Animations Look Choppy
- Use physical device, not simulator
- Close other apps to free memory
- Restart app for fresh state

### Friend Requests Not Showing
- Ensure demo mode is enabled
- Navigate to `/friends` route
- Check that getDemoFriendRequests() returns data
