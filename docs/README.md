# Docs

This repo keeps the long-term docs set small.

- [operations.md](./operations.md): launch runbook, Firebase ops checklist, and App Store resubmission steps.
- [development.md](./development.md): local setup, env handling, EAS/TestFlight build workflow, and developer guardrails.
- [architecture.md](./architecture.md): high-level data, maps, and rules guidance for app and backend changes.
- [app-store.md](./app-store.md): App Store metadata, reviewer notes template, and TestFlight smoke checklist.
- [testflight-readiness-audit.md](./testflight-readiness-audit.md): consumer-launch audit matrix, proxy-only parity checklist, and go/no-go rule before the next build.
- [release-readiness.md](./release-readiness.md): current release tracker for App Store submission and near-term launch work.
- [google-maps-ios-spm-migration.md](./google-maps-ios-spm-migration.md): deferred note on the Google Maps iOS move away from CocoaPods.
- [privacy-policy.md](./privacy-policy.md): public privacy policy copy.
- [terms-of-service.md](./terms-of-service.md): public terms copy.

Stale task handoffs, local key notes, archived audits, and machine-specific setup scraps should not live in the main docs set.
